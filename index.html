What we're building

Leading the engineering delivery of an AI-powered vehicle inspection system for Ryder Systems — one of the largest truck fleet operators in the US. The system replaces manual walk-around inspections with a photo-driven pipeline: a technician photographs the vehicle, the AI scans every image for damage, and a structured digital inspection report is produced automatically. Funded by Microsoft's ECIF programme and built entirely on Azure, with a fixed 9-week delivery window ending mid-June 2026.

Architecture & Pipeline

Designed the end-to-end system architecture on Azure: photo ingestion, a computer vision model that detects scratches, dents, rust, paint chips, and missing parts, a verdict layer that outputs Pass / Fail / Not Applicable per checklist item with annotated images and confidence scores, a generative AI layer that writes human-readable damage descriptions and recommended actions, and a JSON API output that plugs directly into Ryder's existing fleet management systems. Low-confidence detections are automatically routed to a human review queue — keeping humans in the loop for ambiguous cases without slowing down clear-cut ones.

AI Model & Data

Owning the computer vision model training pipeline — sourcing and labelling thousands of vehicle photos across damage categories, fine-tuning the detection model, and building the evaluation harness that measures per-class accuracy across scratch, dent, rust, and cleanliness categories. The checklist is dynamic: it adapts based on what the model finds in each image set, rather than running a fixed list regardless of vehicle condition.

Infrastructure & Delivery

Responsible for the full Azure infrastructure — compute, storage, monitoring, and the secure API layer. Building to a tight external deadline with $93,893 Microsoft ECIF funding at stake: Phase 1 complete by June 12, full system in Ryder's UAT environment by June 17. Scope is one vehicle category end-to-end — from raw photos to a validated, documented, handover-ready system.

Autonomous Bug Resolution System

  • Planner agent reads the JIRA ticket and identifies the affected code and root cause.
  • Developer agent generates the fix with full codebase context awareness.

LLM Evaluation Harness
CI-ready A/B Evaluation Framework

A production-ready framework for A/B evaluating LLM changes with statistical rigour and CI integration — making evaluation a real deployment gate, not a manual check.

  • Hard checks first: Schema, citation presence, format requirements — deterministic, zero cost. Only passing cases proceed to LLM judge.
  • LLM-as-judge with calibration: Rubric-based pairwise scoring with agreement rate, flip rate, and order bias measurement.
  • 95% bootstrap confidence intervals on win rate. Per-tag breakdown. Refusal and format pass rates tracked separately.
  • Quality gates: Configurable thresholds. Exit code 2 on failure blocks CI pipeline. Regression detection vs stored baseline.
  • Zero-cost CI: Mock providers run the full pipeline in GitHub Actions with no API keys.
