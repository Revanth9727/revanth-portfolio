001 Agentic · macOS
RAOC — Remote Autonomous OS Controller
PythonClaude APITelegramSQLitePydantic v2macOS Keychain

RAOC lets you control your Mac from your phone by sending natural language commands over Telegram. It's an exploration of agentic OS control with a rigorous safety model — the system does nothing without your explicit approval.

  • 6-agent pipeline: Each command flows through six specialised agents — intake (understand the request), discovery (find the target file or context), planning (generate a step-by-step execution plan), execution (carry out the plan), verification (confirm it worked), and reporter (send proof back to your phone with screenshots and results).
  • Safety-first architecture: Every action requires tap-to-approve before anything runs. Workspace sandboxing means agents can only touch files inside a designated directory. Dangerous commands (rm -rf, sudo, etc.) are blocked at the substrate layer. Every file is automatically backed up before being modified, with auto-restore on failure. Only your Telegram user ID can issue commands — anyone else is silently ignored.
  • What it can do: Rewrite documents in a specified tone, run data analysis scripts, find files and act on them, execute shell commands — all from a chat message on your phone. ~40% of cached queries return in under 150ms; the rest complete in 2–4 seconds.
  • Engineering quality: 303 passing tests covering all agents, the coordinator, and substrate components. Built with Python 3.12, Claude API (Sonnet for planning, Haiku for narration), SQLite / SQLAlchemy for job tracking, and macOS Keychain for secret storage.

002 Multi-agent
Bugfix AI Pilot
PythonTypeScriptReactDockerGPT-4JIRA APIGitHub API

An autonomous system that closes the loop between a bug report and a merged code fix, with no human in the middle. It reads a JIRA ticket, understands the affected code, generates a fix, validates it, and opens a PR — all automatically.

  • 4-agent architecture: The Planner agent reads the JIRA ticket and identifies the affected code areas. The Developer agent generates code fixes using GPT-4. The QA agent runs automated tests against the proposed fix to validate it works. The Communicator agent updates the JIRA ticket with progress and opens a GitHub PR with the fix and a description of what was changed and why.
  • Full-stack system: Each agent runs as a separate Python microservice. A central FastAPI backend orchestrates the workflow between them. A React / TypeScript frontend provides a live monitoring dashboard showing the status of each ticket as it moves through the pipeline. Docker Compose handles the full multi-service deployment locally.
  • Real engineering depth: 233 commits reflects a project that was built, broken, debugged, and rebuilt — not a weekend demo. The system handles edge cases like test failures (the QA agent loops back to the Developer with failure context), JIRA API rate limits, and GitHub PR conflicts.

003 EvalOps
LLM Evaluation Harness
PythonRAGASGitHub ActionsOpenAIOllamaNumPy

A production-grade, CI-native A/B evaluation framework for LLMs — built to codify the EvalOps practices I use at CGI into a reusable open-source tool. The goal is to make rigorous LLM evaluation as routine as running unit tests.

  • Statistical rigour: Every evaluation run produces win rate with 95% bootstrap confidence intervals — not just a point estimate. You see both the win rate and the uncertainty range, so you know whether an improvement is real or noise. Also tracks refusal rate, format pass rate, and latency per candidate.
  • LLM-as-judge with hard checks first: Deterministic checks (schema validation, citation presence, length constraints) run first and are cheap. Only cases that pass hard checks go to the LLM judge — keeping evaluation costs down. The judge uses structured rubrics and exemplars for calibration.
  • CI/CD quality gates: Configurable thresholds (min win rate, max refusal rate, min format pass rate) that fail the build with exit code 2 if a model update regresses quality. Regression detection compares against a baseline summary — if win rate drops more than a configurable tolerance, the gate blocks deployment.
  • Zero-cost CI: Mock providers simulate model responses so the full evaluation pipeline runs in CI without making real API calls. Supports OpenAI, Ollama (local models), and any OpenAI-compatible API for production runs.
